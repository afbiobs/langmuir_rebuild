## Executive Situation

The project is no longer blocked by missing matched weather, by the CL
supercritical selector, by stale organisation-time resets, by the 0–3 visible
hierarchy, or by lack of dynamic range in the raw CL instability scale.

Those issues have all been audited directly.

The current problem is more specific and more important:

- the physics stack can now generate substantial range
- the real forcing-matched ERA5 cache is populated for all 67 cases
- but the ordering of predictions against observations is still wrong

In other words, WP-06 is no longer suffering from a narrow-attractor failure.
It is suffering from a ranking / mapping failure.

That matters because a model with the right range but the wrong ordering is not
missing "more variance". It is likely predicting the wrong physical quantity,
using the wrong state history, or mapping the right physics to the wrong
observable.

## First-Principles Reading of the Failure

From first principles, if a model has enough range but poor or negative
correlation with observations, the failure must lie in one or more of the
following layers:

1. the observations are not measuring the same scale the model is predicting
2. the model is using the wrong forcing history for the observed structures
3. the model's control variable orders the cases incorrectly
4. the site metadata or local state variables are wrong enough to scramble the
   ordering

The debugging path should now be organised around those four possibilities,
instead of around generic "more variance" repair work.

---

## Most Plausible Physical Explanations

### A. The model may be predicting the wrong observable

The observation table contains visible band spacing from imagery.
The CL solver predicts an onset instability scale, which is then mapped through
coarsening and a visibility proxy.

If the observed quantity is not well approximated by:

`visible spacing ≈ onset width × 2^(n mergers)`

then even a physically correct instability model can have poor correlation.

This is the strongest current conceptual risk.

The existing evidence points this way:

- once coarsening is freed, the model produces large visible ranges
- but those widths do not line up with the observed ranking
- the visible hierarchy itself is not clipping the result

That suggests the issue may be the observation operator, not the hydro solver
alone.

### B. The observed structures may reflect older forcing than the current reset logic captures

The current matched path uses a 6 h lookback with reset rules based on wind
direction change, low-wind shutdown, and rapid speed increase.

That is a defensible engineering closure, but it may still be too short or too
literal for satellite-visible streak fields.

If visible patterns persist after the wind history that formed them, or if the
actual disruption threshold differs from the current rules, the model may be
ranking cases using the wrong segment of the wind history.

This would scramble correlation even if the hydro and coarsening physics were
otherwise reasonable.

### C. The CL control variable may be misordered across sites

The current CL path depends on:

- forcing-to-`Ra` mapping
- forcing-derived Robin closure
- affine drift reduction
- default shear-profile proxy `U'(z) = 1 + z`
- unstratified shallow-lake assumption `S = 0`

Any of these could preserve broad range while still ranking cases incorrectly.

For example:

- depth errors can reorder `2πh / l_cNL`
- wrong `nu_T` or wrong profile shape can reorder `Ra`
- `S = 0` could overstate instability on some lakes and understate it on others
- the assumed shear profile may vary less like reality than the drift profile

That would create negative or weak correlation without necessarily collapsing
range.

### D. The benchmark may mix multiple physical regimes into one target

The 67 cases likely do not all reflect the same structure family:

- young streaks near onset
- mature coarsened Langmuir cells
- tracer-rich visible bands
- possibly non-LC linear wind rows or mixed wind-wave structures

If the benchmark is aggregating physically different observables under one
"spacing" column, a single monotonic mapping from hydro state to visible
spacing may fail even if each subfamily is individually predictable.

This does not justify cherry-picking subsets to manufacture success.
It does justify explicitly checking whether one observation table is mixing
distinct mechanisms.

## Recommended First-Principles Investigation Plan

The next work should not start by tuning coefficients.
It should start by identifying which layer is scrambling rank order.

### 1. Do a strict rank-order audit by layer

For every case, tabulate and compare the rank ordering of:

- observed spacing
- raw onset width `L_inst`
- coarsened mechanical width
- visible comparison width
- wind speed
- `Ra`
- depth
- `u*_water`
- `nu_T_vertical`
- `A_H`
- pattern lifetime

The question is:

At which layer does the ordering become wrong?

If `L_inst` already has the wrong rank sign, the hydro/forcing layer is the
problem.
If `L_inst` is reasonable but visible spacing flips sign, the observation
mapping is the problem.

### 2. Check within-site ordering separately from cross-site ordering

Do not segment the benchmark to claim success.
Do segment it diagnostically to localise the failure.

Specifically test:

- within-lake rank correlation
- cross-lake rank correlation
- same-day or nearby-day pairwise ordering

If within-site correlation is acceptable but pooled correlation is bad, then
site metadata or cross-site physics closures are the main problem.
If within-site correlation is also bad, then the time-history / observation
operator is more suspect.

### 3. Audit the observation operator against image physics

This is probably the highest-value next slice.

Questions to answer:

- Are the measured bands expected to track cell-pair width directly?
- Are they more likely to track tracer-convergence lines than cell width?
- Does one observed band correspond to one cell pair, half a cell pair, or a
  subset of visible streaks within a cell?
- Is the point comparison better represented by the lower edge, upper edge, or
  midpoint of the visible hierarchy?

This should be answered from first principles of what satellite imagery sees,
not from RMSE tuning.

### 4. Extend the state-history audit before changing more hydro physics

The next question is whether the visible structures are older than the current
6 h lookback and reset rules assume.

Test:

- 6 h, 12 h, 18 h, 24 h matched-history windows
- alternative reset thresholds
- persistence of visible structures after low-wind intervals

Do this as a diagnostic matrix, not as an optimisation exercise.

The goal is to learn whether correlation depends primarily on which wind
episode is assumed to have formed the visible pattern.

### 5. Audit site metadata and local depth representativeness

Because spacing scales with depth and shallow-water constraints, a local depth
error can reorder cases strongly.

Check:

- whether the depth used is local to the observed feature or lake-average
- whether fetch and basin geometry are locally representative
- whether the observation coordinates fall near shore, in embayments, or over
  bathymetric transitions where the current metadata are too crude

This is a classic source of anti-correlation in shallow-lake structure models.

### 6. Revisit the hydro closures only after the ranking audit identifies the failing layer

Hydro targets that may still matter:

- forcing-to-`Ra` closure
- shallow-lake shear-profile representation
- role of stratification in the onset map
- Robin closure sensitivity across lakes

But these should now be attacked with a ranking target in mind.

Do not make another hydro change simply because a single metric might improve.
