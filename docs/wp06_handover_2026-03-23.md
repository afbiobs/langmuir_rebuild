## What WP-06 Requires

From `docs/clean_room_rebuild_brief.md`, WP-06 requires:

1. Run the formal candidate comparison on all benchmark subsets.
2. Produce comparison outputs under `outputs/comparison/`.
3. Select the production candidate and document the rationale.
4. Clean up the winning candidate into a production pipeline.
5. Verify that the production pipeline reproduces the comparison numbers exactly.

The brief expects the following output family:

- `metrics_table.csv`
- predicted-vs-observed plots
- spacing-vs-wind plots
- dynamic-range and tail-coverage comparison plots
- attractor diagnostics
- representative case diagnostics

At present, the clean-room harness already produces the core tables and JSON
summaries, but not the full WP-06 plot/output package yet.

## Entry Assumptions for the Next Session

Proceed under these explicit assumptions:

1. The operational shallow-lake onset path remains unstratified (`S = 0`).
2. That assumption is acceptable for WP-06 progression, but must remain visible
   in documentation and interpretation.
3. If a case is materially stratified, the present solver may predict onset
   widths that are too narrow and onset thresholds that are too low.
4. Do not add an ad hoc stratification correction just to improve fit.

This means:

- Stratification is a flagged uncertainty.
- It is not a reason to stop WP-06.
- It is a reason to keep candidate-selection language disciplined.
